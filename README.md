# 프로젝트 이름
숙면에 재미를 더하다, 숙면용사
## 프로젝트 설명
숙면용사는 **재미있는 경험을 통해 수면시간을 자연스럽게 늘리는 게임형 수면 코치 앱** 입니다.

현대인에게 부족한 수면시간, 그 이유는 수면시간이 재미를 경험할 수 없는 시간이기 때문입니다.

숙면용사는 폰을 내려놓고 자는 동안 캐릭터가 보상을 획득해 성장하여

사용자는 “더 오래 수면할수록 더 많이 성장”하는 구조를 경험하게 해줍니다.
## 프로젝트 팀원 소개
- PM - 필립/박준형
- Design - 엠제이/김민주
- Android - 칼리/박준성, 베르/배준혁, 테오/조경민
- SpringBoot - 제로/박준영, 짭상혁/김상혁, 이안/유용선, 루이/최종건

## 이슈 컨벤션
- 제목 형식: [이름/타입] 구현내용

## 브랜치 전략

1. 기본 전략으로 Github Flow를 따릅니다.
- main: 항상 배포 가능한 안정 버전
- feature/*: 기능 단위 개발 브랜치
- fix/*: 수정 작업 브랜치

2. 브랜치 생성 규칙
```bash
├─main
  ├─ lui/feature/jwt-login
  ├─ lui/refactor/user-mypage
```
- 브랜치 이름은 `닉네임/분야/세부내용`을 따릅니다.
- 추후 pr시 main브랜치로 pr을 날리며 main브랜치는 배포용 브랜치라 정말 급한 일이 아니면 여기로 직접 push하면 안됩니다. 
- pr이 완료되면 만든 브랜치는 삭제합니다.

## 커밋 컨벤션
| Message  | 설명                                              |
| :------: | :------------------------------------------------ |
|   feat   | 새로운 기능 추가                            |
|  fix   | 버그 수정 |
|  refactor  | 코드 리펙토링 |
| comment | 주석 추가 및 변경                                     |
|  rename   | 파일 혹은 폴더명 수정                |
|  remove   | 파일 혹은 폴더 삭제                |
|  chore   | 기타 변경사항                |

- 커밋 형식: [이름] 작업 내용
- 예시: [Kalli] Feat: 사용자 로그인 기능 추가
- 여러 커밋 형식이 필요한 경우 (ex: 기능 추가, 리팩토링, 버그 수정을 한 꺼번에 한 경우) 가장 큰 작업 내용으로 컨벤션을 하나만 작성합니다.

## 환경 변수
